# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - 오늘은 자바 쓰레드 모델에 대해서 자세히 학습하는 시간을 가졌다. 
    - Java Thread 클래스, start() 메소드를 통한 JVM의 멀티쓰레딩 과정, Runnable 인터페이스 및 특징
    - 자바에서 쓰레드를 만드는 방법 2가지 (Thread 클래스 상속, Runnable 인터페이스 구현), Thread와 Runnalble의 한계
    - Callable/Future 인터페이스 개념 및 단점
    - Executors, Executor, ExecutorService 의 역할 및 책임.
    - execute()와 submit()의 차이
- (오늘의 목표 중에서) 아직 끝내지 못한 부분:
  - fork, join
  - CompletableFuture
  - JVM의 가상 쓰레드

###### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단:
  - 오늘은 학습만 진행하였기에 가장 의미있었던 학습 과정에서의 판단은 자바 버전 마다의 멀티 쓰레드를 제공하기 위한 인터페이스가 왜 나왔는지, 특징은 무엇인지에 대해서 학습한 것
- 그렇게 판단한 이유:
  - 자바 5이전의 Thread와 Runnable 인터페이스에서 어떤 단점과 불편한 점이 있었기에 Callable, Future 인터페이스가 나왔는지. 그리고 더 나아가 Executors, Executor, ExecutorService의 탄생 이유도 찾아보며 학습하다보니 전반적인 자바 쓰레드 모델의 발전사는 물론이고 책임 분리와 비동기 과정 어떤 점들을 고려해야 하는 지 (예를 들어, Future.get()을 사용하게 된다면 호출한 쪽에서 블로킹이 된다는 문제)를 자세히 알 수 있었다
- 고려했지만 선택하지 않은 대안:
  - 사실 간단하게 알아보고 바로 코딩을 할 수 있었지만, WAS 의 특성 상 다중 요청을 처리하기 위해서 멀티쓰레딩 환경은 필수적이고, 멀티쓰레딩 환경을 구축하는 과정에서 어떤 고민을 했냐에 따라서 WAS의 성능이 달라질 것이라고 판단을 내렸다. 그래서 섣부른 코딩보다는 자바 쓰레드 모델의 발전사에 대해서 꼼꼼히 학습하고자 했다
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향:
  - 다소 시간이 걸리다보니 코드를 작성하지 못했다는 문제점이 존재하지만, 멀티 쓰레딩 환경에서 고려해야 되는 점들을 다소 덜 놓칠거라 기대가 된다

###### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상:
  - ExecutorService의 shutdownNow()를 호출한 경우 자동으로 쓰레드 풀 작업 큐에 들어있던 작업들이 진행되지 않고 바로 종료될 줄 알았다. 하지만 shutdownNow()를 호출해도 큐에 들어있는 작업들이 계속해서 진행되는 예제 코드가 이해가 되지 않았다
- 처음 추측한 원인:
  - 솔직히 잘 이해가 되지 않았다. cs를 배울 때 인터럽트가 발생하면 진행되던 작업이 바로 중단되는 것으로 알고 있었는데, 그렇지 않았기에 왜 그런지 이유를 고민해 보았으나 알 수 없었다
- 원인을 확인하기 위해 시도한 방법:
  - shutdownNow()의 내부 동작을 뜯어보았다
- 실제 원인:
  - 실제 쓰레드를 멈추는것이 아니라 인터럽트 플래그를 true로 바꾸는 것이며, 작업들에서 인터럽트를 다루는 코드가 없으면 작업은 중단되지 않고 계속 진행되는 것이였다
- 해결 방법:
  - Task를 작성할 때 인터럽트가 발생한 경우에 대한 처리 코드를 작성한다. 
- 다음에 비슷한 문제가 생기면 먼저 확인할 것:
  - 인터럽트 처리 코드가 작성되어 있는지 확인하고, 디버깅을 통해 인터럽트 코드가 잘 작동하는지 확인한다

###### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지:
  - 자바 쓰레드 모델의 발전사를 공부할 때 인터페이스와 클래스들의 장단점, Runnable과 Callable을 어떨 때 써야하는지, 각 인터페이스들의 메소드를 사용할 때 주의할 점은 무엇인지에 대해서 찾아보고 공부한 행동
- 그 접근이 도움이 된 이유:
  - 클래스와 인터페이스가 이전 자바 버전있던 어떤 문제를 해결하기 위해서 나왔는지에 대해서 이해할 수 있었으며, 메소드 사용법에 대한 주의점에 대해서 찾아보며 멀티 쓰레딩 환경에서의 개발 방법에 대한 기초를 닦을 수 있었다
- 다음에도 반복하고 싶은 점:
  - 왜 이 기술이나 인터페이스, 클래스가 나왔는지에 대해서 꼼꼼히 공부하고 싶으며, 이에 따른 각각의 장단점 그리고 어떤 상황에서 사용할 수 있는지 나만의 언어로 정의하는 것을 지속하고자 한다

###### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점:
  - 개발을 하지 못해서 아쉬움이 남는다. 이는 쓰레드 모델에 대해서 공부한 시간이 생각보다 오래 걸려서 그렇다
- 그렇게 된 원인:
  - 하나의 글을 읽을 때 잘 이해가 되지 않았어도 끝까지 읽은 후에 다시 한번 이해하려고 했어야 하는데, 그 다음 글로 넘어가서 다시 처음으로 돌아와 다시 읽고 이해하는 과정을 거친게 원인이다. 
- 다음에는 다르게 해볼 구체적인 행동:
  - 전반적인 흐름을 파악하고 이해하려고 하였지만, 이 또한 앞 내용이 어느정도 이해가 되어야 흐름을 잘 이해한다고 생각한다. 그렇기에 다음에는 기술 블로그 글을 하나 읽은 뒤에는 내 머릿속으로 좀 정리하는 시간을 갖고자 한다. 추가적으로 잘 이해가 안되는 부분은 LLM을 통해 예시 코드를 보며 좀 더 빠르게 이해하고자 한다
